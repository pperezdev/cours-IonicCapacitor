Configurez un projet par défaut et prenez un peu de temps pour examiner son architecture.

Essayez de générer un autre projet en utilisant un template différent, puis comparez le contenu de chacun ainsi que leurs différences.

Comme vous êtes habitué à React, vous devriez pouvoir repérer facilement toutes les implémentations qui en sont très proches.

2.

En utilisant vos connaissances en React et le template de base des projets, essayez de manipuler le code existant.

Voici une liste de fonctionnalités que vous pouvez déjà implémenter grâce à vos connaissances en React :

* Appelez une API pour récupérer une liste d’éléments (par exemple, l’API GitHub pour récupérer vos dépôts) :
    https://api.github.com/users/“Votre-Utilisateur”/repos
* À partir de cette requête, vous pouvez déjà créer une page qui récupère les résultats et affiche chaque dépôt avec son nom, son image, sa date, son nombre d’étoiles et d’autres informations.
* Essayez de présenter les résultats sous forme de liste.
* Ensuite, implémentez une solution simple permettant à l’utilisateur de saisir un nom d’utilisateur GitHub. Cela devra déclencher l’appel HTTP et afficher le résultat, ou une erreur si le nom d’utilisateur n’existe pas.
* En utilisant tous les composants fournis par Ionic, essayez de proposer une interface agréable, à la fois pour afficher les résultats mais aussi pour gérer les erreurs.
* Ne vous limitez pas aux composants présentés dans les slides, utilisez également ceux disponibles dans la documentation.
